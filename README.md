# IQtools

一个基于 **C++** 与 **Qt 6** 开发的桌面工具箱应用。

## 项目简介

IQtools 旨在构建一个轻量、易扩展、界面友好的桌面工具集合，用于集中提供日常开发、调试与效率辅助功能。项目当前处于起步阶段，后续将逐步完善整体架构、模块设计与具体工具能力。

## 技术栈

- **语言**：C++
- **框架**：Qt 6
- **类型**：桌面应用 / 工具箱应用

## 规划目标

- 提供统一的工具入口与清晰的模块化界面
- 支持常用小工具的快速集成与扩展
- 保持良好的可维护性与跨平台能力
- 逐步完善基础框架、功能页面与交互体验

1. 项目基础结构搭建

## 核心能力规划

第一阶段计划围绕以下基础能力展开：

- **截图工具**：支持区域截图、窗口截图、全屏截图，以及后续截图结果处理能力
- **翻译工具**：支持文本翻译、截图后文字识别与翻译等扩展场景
- **自定义插件式工具扩展**：支持后续按模块扩展更多独立工具能力

在此基础上，后续还可以逐步加入：

- 文本处理工具
- 编码/解码工具
- JSON / XML 格式化工具
- 时间戳转换工具
- 网络调试辅助工具

## 插件功能技术方案建议

对于 Qt 6 + C++ 的桌面应用，当前最推荐的插件实现方式是：

- **优先采用 Qt 原生插件机制**
- 推荐技术组合：`QPluginLoader + 接口类 + JSON Metadata`

### 为什么推荐这个方案

- 与 Qt 6 生态兼容性最好，适合桌面应用长期维护
- 插件与主程序都基于 C++ / Qt，类型系统、对象生命周期和调试方式更统一
- 跨平台支持更成熟，适合 Windows / macOS / Linux 后续扩展
- 比自定义动态库协议更稳定，能减少重复造轮子
- 更适合截图、翻译这类需要较深 UI 集成和系统能力的模块

### 分阶段建议

#### v1

- 使用 **Qt C++ 动态插件** 作为唯一插件方式
- 建立统一插件目录、插件发现与加载机制
- 先定义最小插件接口，例如：插件信息、初始化、关闭、工具入口

#### v2

- 将插件分层为：
  - **Tool Plugin**：用户可直接使用的工具模块
  - **Provider Plugin**：翻译引擎、OCR 引擎等服务提供者
- 增加 Host API，向插件提供日志、配置、网络、任务调度等能力

#### v3

- 再考虑增加脚本型插件或进程隔离机制
- 用于第三方扩展、自动化能力、插件市场等更复杂场景

### 职责划分建议

- **主程序负责**：插件管理、统一入口、主界面框架、权限处理、公共能力服务
- **截图工具**：建议优先作为核心模块或核心插件实现，主程序掌握截图会话与系统集成
- **翻译工具**：建议将界面与流程统一管理，将具体翻译引擎做成可替换的 provider 插件
- **第三方扩展**：在稳定插件接口后，逐步开放给自定义插件开发

## 基础系统规划

### 日志系统

项目将内置统一日志系统，作为应用内核的基础能力之一，主要用于：

- 记录应用启动、关闭、模块初始化等生命周期事件
- 记录截图、翻译、插件加载等核心流程运行状态
- 记录警告、异常与错误信息，便于调试和问题定位
- 为后续插件系统提供统一日志接口

第一阶段日志系统的目标：

- 支持控制台输出
- 支持文件输出
- 支持日志级别划分（Debug / Info / Warning / Error）
- 支持统一日志格式
- 支持主程序与插件共享同一套日志能力

建议目录位置：

- `infra/logging/`：日志底层实现
- `core/services/`：对日志能力进行统一暴露

后续可继续增强：

- 日志文件滚动归档
- 分类日志与模块标签
- 插件日志隔离与聚合
- 崩溃前关键日志落盘
- 可视化日志查看页面
